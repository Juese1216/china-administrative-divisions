# 13 类关系例句

本表中的例句来自 `data/processed/relation_details/triples.csv` 的真实抽取结果，用于说明“13 类关系到底是什么”。

## 例句总表

| 关系代码 | 中文关系 | 真实例句 | 关系示意 | 类型说明 |
| --- | --- | --- | --- | --- |
| `REVOKE_ADMIN` | 撤销建制 | 撤销福建省莆田县。 | 莆田县 -> 撤销建制 -> 撤销事件 | 表示某个已有行政区划被撤销。 |
| `ESTABLISH_ADMIN` | 设立建制 | 设立县级北屯市。 | 北屯市 -> 设立建制 -> 设立事件 | 表示新设立一个行政区划单位。 |
| `RENAME_ADMIN` | 名称变更 | 潞西市更名为芒市。 | 潞西市 -> 更名为 -> 芒市 | 表示行政区划名称发生变化。 |
| `MERGE_ADMIN` | 合并建制 | 将宁冈县和县级井冈山市合并，组建新的县级井冈山市。 | 宁冈县 / 井冈山市 -> 合并为 -> 新井冈山市 | 表示多个行政区划合并成新的行政区划。 |
| `TRANSFER_ADMIN` | 区域划转 | 将六安市寿县划归淮南市管辖。 | 寿县 -> 划归 -> 淮南市 | 表示某个行政区划或区域划入另一个行政区管辖。 |
| `JURISDICTION_ADMIN` | 管辖隶属 | 汤旺县由伊春市管辖。 | 汤旺县 -> 隶属于/由...管辖 -> 伊春市 | 表示调整后的上下级管辖关系。 |
| `DIRECT_ADMIN` | 省级直辖 | 共青城市由江西省直辖。 | 共青城市 -> 直辖于 -> 江西省 | 表示县级市等单位由省级行政区直接管辖。 |
| `ENTRUST_ADMIN` | 委托代管 | 将资阳市代管的县级简阳市改由成都市代管。 | 简阳市 -> 代管于 -> 成都市 | 表示一个行政区由另一个城市代管。 |
| `GOV_RESIDENCE` | 政府驻地 | 区人民政府驻科尔沁大街102号。 | 区人民政府 -> 政府驻地 -> 科尔沁大街102号 | 表示政府机关驻地在哪里。 |
| `RESIDENCE_TRANSFER` | 驻地迁移 | 龙湾区人民政府驻地由状元镇迁至永中镇。 | 龙湾区 -> 驻地迁至 -> 永中镇 | 表示政府驻地从一个地点迁到另一个地点。 |
| `AREA_INHERITANCE` | 行政区域承继 | 撤销北京市昌平县，设立北京市昌平区，以原昌平县的行政区域为昌平区的行政区域。 | 昌平区 -> 承继行政区域 -> 原昌平县行政区域 | 表示新区划继承原区划的行政区域范围。 |
| `ADJUSTMENT_EVENT` | 行政区划调整事件 | 调整河北省廊坊市安次区的行政区划，增设廊坊市广阳区。 | 安次区 -> 发生行政区划调整 -> 调整事件 | 表示一条综合性的行政区划调整事件，后面常伴随设立、划转等子关系。 |
| `SCOPE_CONSTRAINT` | 范围包含排除 | 撤销湖北省宜昌县，设立宜昌市夷陵区，以原宜昌县的行政区域（不包括土城乡、桥边镇、艾家镇）为夷陵区的行政区域。 | 宜昌县行政区域 -> 不包括 -> 土城乡/桥边镇/艾家镇 | 表示行政区域范围里的包含、排除或边界限制。 |

## 类型说明示例

可以这样说：

> 这 13 类关系不是随便列的，而是根据行政区划变更文本中常见的动作和句式归纳出来的。比如“撤销”“设立”“更名”“划归”“管辖”“驻地迁至”等都有比较明确的触发词；“行政区域承继”和“范围包含排除”则用于表达区域范围的继承和边界约束。

关于“为什么一条记录会有多个标签”，可以用这个例子：

> 例如“撤销北京市昌平县，设立北京市昌平区，以原昌平县的行政区域为昌平区的行政区域。”这一条同时包含“撤销建制”“设立建制”和“行政区域承继”三个关系。因此分类阶段做成 13 类多标签，而不是只选一个类别。

## 最容易混淆的几组

| 容易混淆的关系 | 区别 |
| --- | --- |
| `GOV_RESIDENCE` 和 `RESIDENCE_TRANSFER` | `GOV_RESIDENCE` 只是说明政府驻在哪里；`RESIDENCE_TRANSFER` 强调“由 A 迁至 B”的迁移过程。 |
| `TRANSFER_ADMIN` 和 `JURISDICTION_ADMIN` | `TRANSFER_ADMIN` 强调“划归/划入”的调整动作；`JURISDICTION_ADMIN` 强调调整后的管辖关系。 |
| `DIRECT_ADMIN` 和 `ENTRUST_ADMIN` | `DIRECT_ADMIN` 是省级直辖；`ENTRUST_ADMIN` 是由某个城市代管。 |
| `AREA_INHERITANCE` 和 `SCOPE_CONSTRAINT` | `AREA_INHERITANCE` 表示新区划继承原行政区域；`SCOPE_CONSTRAINT` 表示继承或划转时“不包括/不含/包括”的范围限制。 |
| `ADJUSTMENT_EVENT` 和其他具体关系 | `ADJUSTMENT_EVENT` 是综合事件标签，具体动作通常还会由设立、撤销、划转等关系进一步表达。 |
